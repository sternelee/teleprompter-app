# 项目与交互流程分析报告

## 1. 项目概述

- **产品**：AI 英语口语练习提词器（Teleprompter）。
- **核心闭环**：用户描述真实场景 → DeepSeek 生成对话 → 语音实时识别 → 逐词高亮进度 → 自动滚屏 + 错题提示 → 接近结尾时自动续写对话。
- **技术栈**：Expo SDK 56、React Native 0.85.3、React 19.2.3、TypeScript 6.0、expo-router、expo-speech-recognition。
- **目标平台**：iOS、Android、Web（static output）。
- **设计方向**：Animal Crossing / NookPhone 风格，暖色羊皮纸背景、薄荷青主色、圆润气泡、3D 按压阴影。

- **状态管理**：单一 React Context，无外部状态库；UI 状态用 `useState`，可变值用 `useRef`。
- **导航**：`expo-router` 文件路由；`_layout.tsx` 注册 3 个 Stack 页面，其中 `settings` 为 modal。

## 4. 全局状态（AppContext）

| 字段 | 类型 | 说明 |
|------|------|------|
| `apiKey` | `string` | DeepSeek API Key，仅内存存储，页面刷新丢失 |
| `scene` | `string` | 用户描述的场景 |
| `segments` | `DialogueSegment[]` | AI 生成的对话段落 |
| `isGenerating` | `boolean` | 生成中标志 |
| `generationError` | `string \| null` | 生成错误信息 |
| `appendSegments` | `function` | 自动续写时追加段落 |
| `resetSession` | `function` | 清空场景与对话 |

- 所有 setter 都用 `useCallback` 包装，避免子组件非必要重渲染。
- `apiKey` 通过 `secureTextEntry` 输入，不持久化，符合“单次会话”安全模型。

## 5. 页面交互流程

### 5.1 首页 / 三步向导（index.tsx）

| 步骤 | 关键内容 | 校验 |
|------|----------|------|
| **Scene** | 多行输入 + 4 个快速场景 chips | 词数 ≥ 9 为 Ready；≥ 5 为 Almost；否则提示补充 |
| **Key** | 展示 API Key 是否已填写 | 未填写可跳 Settings |
| **Review** | 汇总场景与 Key 状态 | 两者 Ready 才可生成 |

- **主按钮逻辑**：
  - Scene 页：输入有效 → Key 页。
  - Key 页：无 Key → 跳转 `/settings`；有 Key → Review 页。
  - Review 页：调用 `generateDialogue(scene, apiKey)` → 写入 `segments` → 跳转 `/teleprompter`。
- **错误处理**：缺少 Key / 缺少场景 / API 报错都会在当前页底部以红色卡片展示，并自动切到对应 tab。
- **响应式**：宽度 `< 640` 时切换为紧凑布局，按钮垂直堆叠。

### 5.2 设置页（settings.tsx）

- Modal 形式从底部/右侧弹出。
- 仅一个输入框：`secureTextEntry` 隐藏 Key。
- 说明文案强调“仅内存存储，只发往 DeepSeek”。
- 点击 Done 返回上一页。

### 5.3 提词器练习页（teleprompter.tsx）

#### 核心数据转换

- `segments` → `allWords`：把每段文本按空白切词，标注 `globalIndex / segmentIndex / localIndex / isSpoken`。
- `wordsBySegment`：按段落分组，用于气泡渲染与 cue 预览。
- `practiceWords` 当前等于 `allWords`（整段对话都需要练习）。

#### 语音监听

- 通过 `useSpeechRecognition` 监听，得到连续转写文本。
- `onResult` 调用 `updateProgress(transcript)`，触发词匹配。
- 点击主按钮：
  - 未开始 → 请求麦克风权限 → `startListening`。
  - 进行中 → `stopListening` → 暂停。
- 离开页面/卸载时自动 `stopListening`。

#### 词匹配与进度

- `useWordMatcher`：
  - 在当前位置前后回溯 3、向前看 20 的窗口内搜索最匹配序列。
  - 使用 Levenshtein 相似度，阈值 `0.6`。
  - 连续匹配得分加权。
  - 跳过的词（长度 ≥ 2 且不在 spoken 文本中）记入 `corrections`。
- 进度条：`spokenCount / totalWords * 100%`。

#### 自动滚屏

- 记录每个 segment 的 `layout.y` 偏移量。
- 当前词变化时计算目标 `targetY`：
  - 若在当前段落内，按段内局部进度插值。
  - 否则按全局进度百分比。
  - 最终向上偏移 `viewportHeight * 0.18`，保证当前词位于视口偏上位置。

#### 教练面板（Coach Panel）

- 默认折叠，点击展开。
- 折叠态显示：状态 pill + 一句话详情 + 已完成/总数。
- 展开态显示：
  - 场景原文。
  - 三指标：Practiced / Left / Revisit。
  - Current focus / Coming up 两句 cue 预览。
  - 操作提示：点击跳转，长按查 Cambridge Dictionary。

#### 单词交互

- **单击**：`jumpToWord(globalIndex)`，若正在监听则先 stop、跳转后再 start，实现无缝续练。
- **长按**：打开 Cambridge Dictionary 对应单词页；使用 `expo-web-browser` 内置浏览器。

#### 自动续写

- 触发条件（`useEffect`）：
  - 有 scene 和 apiKey；
  - 当前处于最后一段（`isOnFinalSegment`）；
  - 已有练习进度（`spokenCount > 0`）；
  - 未在生成/续写中；
  - 对同一段未续写过（`lastAutoContinueSegmentIdRef`）。
- 调用 `generateDialogue(scene, apiKey, segments)` 追加新段落。
- 错误以 `autoContinueError` 展示，不影响继续练习。

> 注意：当前实现以“进入最后一段”作为触发条件，而不是 README 中提到的“剩余词数 ≤ 8”。这会导致用户刚念到最后一段开头时即触发续写，可能过早。

#### 控制区

- **Start / Resume / Pause**：主按钮，颜色随状态变化。
- **Restart**：重置进度、停止监听、滚回顶部。
- **Back**：停止监听并返回首页。
- **错题提示**：底部显示最近 3 个需要复习的词。

## 6. 关键 Hooks 详解

### 6.1 useSpeechRecognition

- 封装 `expo-speech-recognition`。
- 配置：`continuous: true`、`interimResults: true`、`lang: en-US`。
- 事件：`start / result / end / error`。
- 自动重启：`end` 事件后 250ms 重新 `startListening`，保证长时间练习不中断。
- 权限：首次启动时请求；返回 `permissionResponse` 供 UI 判断。

### 6.2 useWordMatcher

- `normalizeWord`：小写 + 去除标。
- `wordSimilarity`：基于最小编辑距离，归一化到 `[0, 1]`。
- `updateProgress`：
  - 忽略与上一次完全相同的转写文本。
  - 在搜索窗口内枚举起始位置，计算匹配得分。
  - 更新 `currentWordIndex` 为最佳匹配末尾词。
  - 对未匹配到的跳过词生成 `Correction`。
- `jumpToWord`：支持点击跳词；会清除目标位置之后的 corrections。
- `resetProgress`：清空所有匹配状态。

## 7. 服务端生成（services/openai.ts）

- 真实调用 **DeepSeek**（不是 OpenAI）：
  - Endpoint：`https://api.deepseek.com/chat/completions`
  - Model：`deepseek-chat`
  - Auth：`Bearer {apiKey}`
  - 启用 `response_format: { type: 'json_object' }`
- Prompt 规则：
  - 首次生成 10-14 行（5-7 轮）。
  - 续写时 8-12 行（4-6 轮），并附加历史对话上下文。
  - 要求保留角色、场景、语气与未完成目标。
  - 返回格式固定为 `{ segments: [{speaker, text}] }`。
- 后处理：
  - 去除 markdown fence。
  - 过滤空文本。
  - `speaker` 只识别 `user`，其余均归 `ai`。
  - 校验至少包含一条 `user` 行，否则抛错。
  - 为每段生成 `id: {timestamp}-{random}`。

## 8. UI 组件

### 8.1 ThemedText / ThemedView

- `ThemedText`：按 `type` 提供 10 种预设样式，支持 `themeColor` 与 `weight`。
- `ThemedView`：按 `type` 设置背景色。
- 主题来自 `useTheme()`，但当前多处样式硬编码 `Colors.light.*`，暗色模式未完全适配。

### 8.2 TeleprompterDisplay

- 每段对话渲染为一个气泡：
  - `user`：NookPalette 循环色，白色文字。
  - `ai`：奶油色背景，棕色文字。
- 词状态：
  - `spoken`：绿色 `#6fba2c`。
  - `current`：白色高亮背景。
  - `correction`：红色下划线。
- 当前段带 `Speak now` / `Reviewed` badge；非当前段略微变暗。
- 通过 `onSegmentLayout` 上报每段 Y 坐标，用于自动滚屏。

## 10. 优势

1. **闭环完整**：从场景输入到语音练习、自动续写、错题复习，链路清晰。
2. **模糊匹配鲁棒**：Levenshtein + lookahead/backtrack 能容忍连读、弱读、识别误差。
3. **交互细节到位**：自动滚屏、单词点击跳转、长按查词、状态 pill 呼吸灯。
4. **视觉风格统一**：Animal Crossing 设计系统贯穿按钮、气泡、颜色、阴影。
5. **代码组织清晰**：按职责分层（page / hook / service / component / context）。

## 11. 潜在问题与风险

| # | 问题 | 影响 | 建议 |
|---|------|------|------|
| 1 | 暗色模式未完整适配：大量样式硬编码 `Colors.light.*`，`teleprompter.tsx` 容器背景写死 `#f6f3e8` | 系统切到深色时界面错乱 | 全部改用 `useTheme()` 或 `ThemedView` 主题类型 |
| 2 | 自动续写触发过早：以“进入最后一段”为条件，而非 README 声称的“剩余 ≤ 8 词” | 用户刚开始最后一段就请求新对话，浪费 Token | 改为 `remainingWords <= 8` 且 `spokenCount > 0` |
| 3 | `apiKey` 仅存内存，刷新即丢 | 每次重新输入，体验略繁琐 | 评估是否需要安全存储（如 Keychain/Keystore）或会话缓存 |
| 4 | 无测试框架 | 回归成本高 | 添加 Jest / React Native Testing Library |
| 5 | `expo-speech-recognition` Web 支持不确定 | Web 端练习功能可能不可用 | 在 Web 增加降级提示或检测 |
| 6 | `AnimatedIcon` / `AnimatedSplashOverlay` 引用本地图片 `logo-glow.png`、`expo-logo.png` | 若未放置对应资源会编译/运行报错 | 删除未使用组件或补全资源 |
| 7 | 所有对话词都作为 `practiceWords`，包括 AI 台词 | 用户实际只需要练习自己的台词 | 支持仅高亮 `user` 段，或提供角色切换 |
| 8 | `corrections` 只记录漏词，不记录发音错误/替换词 | 纠错维度有限 | 可记录 `actual` 为识别到的近似词 |
| 9 | 长段对话时 `allWords` 每次 render 重新 `flatMap` | 大数据时轻微性能开销 | 用 `useMemo` 已做，但 dependencies 包含 `segments`，追加后续写会重建 | |
| 10 | 语音 `result` 事件使用 `event.results[0]` 或最后一条，可能拿到完整历史转写 | 匹配窗口计算基于完整文本，通常无害，但需确认平台行为 | 根据平台差异处理 interim/final |

## 12. 改进建议（优先级排序）

1. **修复暗色模式**：统一使用主题 token，移除硬编码色值。
2. **修正自动续写条件**：`remainingWords <= 8 && spokenCount > 0`。
3. **增加测试**：至少覆盖 `useWordMatcher`、`generateDialogue` 解析、`teleprompter-display` 渲染。
4. **API Key 可选持久化**：使用 `expo-secure-store` 加密存储，并支持清除。
5. **Web 降级**：检测 `ExpoSpeechRecognitionModule` 可用性，不可用时提示用户。
6. **角色练习模式**：允许用户只练习 `user` 台词，AI 台词只用于上下文。
7. ** richer corrections**：记录用户实际说出的词、相似度、时间戳，用于复习列表。
8. **加载与空状态**：生成对话时显示骨架屏；无麦克风权限时提供系统设置入口。
9. **清理未引用组件**：移除或修复 `animated-icon.tsx` 等资源引用。
10. **性能**：在 `TeleprompterDisplay` 中对长列表使用 `React.memo` 或虚拟化（如对话很长）。

## 14. 结论

项目已具备可运行的英语口语练习 MVP，数据流、状态管理、语音匹配、自动滚屏、自动续写等核心链路完整。主要改进空间集中在 **暗色模式适配**、**自动续写触发时机**、**测试覆盖** 与 **Web/权限降级处理**。优先修复前两项即可显著提升产品稳定性。
