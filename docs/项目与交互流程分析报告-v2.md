# 项目与交互流程深度分析报告（v2）

## 视角说明

本报告从 **产品定位、用户旅程、交互细节、用户体验、技术实现** 四个维度重新审视项目，不只做代码层罗列，更关注“用户在使用时会遇到什么、期待什么、哪里会卡住”。

---

## 1. 产品定位再梳理

### 1.1 核心价值主张

| 维度 | 当前定位 | 可延伸价值 |
|------|----------|-----------|
| 用户 | 中级英语学习者 | 可扩展至备考雅思/托福口语、面试模拟、商务英语 |
| 场景 | 日常生活对话（咖啡、酒店、看病等） | 可扩展至演讲稿、会议发言、面试自我介绍 |
| 模式 | 单人跟读 AI 对话 | 可加入 AI 语音播报、角色切换、语速调节 |
| 数据 | 单次会话，刷新即失 | 可本地保存历史、进度、错题本 |

### 1.2 当前产品闭环

```
发现场景 → 描述场景 → AI 生成对话 → 语音练习 → 实时反馈 → 自动续写 → 复习错题
```

闭环已跑通，但缺少“留存”和“成长感”：
- 用户每次进入都是“从零开始”。
- 没有历史记录，无法回顾练过的主题。
- 没有错题本的长线积累。
- 没有成就/进度可视化。

## 2. 用户旅程地图（User Journey Map）

| 阶段 | 用户目标 | 当前体验 | 痛点 | 机会点 |
|------|---------|---------|------|--------|
| **发现** | 想练英语口语 | 打开 App 看到三步向导 | 价值传递快，但缺少示例演示 | 增加 5 秒动画 demo |
| **配置** | 填入 API Key | Settings modal | 每次启动都要重填 | 本地安全存储 |
| **输入场景** | 描述想练的情景 | 多行输入 + 4 个快捷场景 | 快捷场景太少，无法自定义保存 | 历史场景 + 收藏 |
| **生成对话** | 拿到自然对话 | 等待 1-5 秒 | 没有加载动画，网络慢时焦虑 | 骨架屏 + 生成进度提示 |
| **开始练习** | 跟读台词 | 主按钮 Start Speaking | 第一次不知道怎么用 | 新手引导 / 首次 tooltip |
| **练习中** | 被实时高亮反馈 | 词变绿、当前词高亮、自动滚屏 | 暗色模式错乱、当前词音频缺失 | 修复主题、加低调音频按钮 |

| **练完后** | 有成就感 | 仅显示完成 | 无总结、无历史 | 会话报告 + 历史列表 |

## 4. 交互细节逐项分析

### 4.1 首页三步向导（index.tsx）

#### 问题 1：Tab 高亮背景圆角不匹配

**现象**：
- `tabBar` 容器 `borderRadius: Radius.lg`（24）。
- `tabItemActive` 背景色块使用 `borderRadius: Radius.base`（18）。
- 视觉上 active 背景与外层容器圆角不一致，产生“缺角”感。

**修复建议**：
```ts
// styles.tabItemActive
borderRadius: Radius.base, // 18
// 建议改为与容器内边距匹配的圆角，或使用 overflow:hidden 让 tabItem 贴合容器
```

更优方案：在 `tabBar` 上加 `overflow: 'hidden'`，`tabItem` 圆角统一为 `Radius.base` 或 `Radius.sm`，使 active 块边缘与容器视觉对齐。

#### 问题 2：步骤切换可点击但缺少完成校验

**现象**：用户未输入场景时，点击 Step 2/3 可切换，但内容为空或报错。

**建议**：
- 未完成前置步骤时，Tab 仍可点击但显示引导文案（如“请先描述场景”）。
- 或禁用未解锁步骤，增加锁图标。

#### 问题 3：快速场景 chips 不支持自定义保存

**建议**：
- 用户输入过的场景自动出现在“最近使用”。
- 支持左滑收藏 / 删除。

### 4.2 API Key 管理

#### 问题：仅存内存，每次启动丢失

**影响**：
- 高频打断：每次打开 App 都要重新输入 32-64 位 Key。
- 安全感误解：文案说“仅内存存储”，用户可能以为更安全，但实际上降低留存。

**建议方案**：

| 方案 | 实现 | 安全性 | 体验 |
|------|------|--------|------|
| `expo-secure-store` | 加密写入 iOS Keychain / Android Keystore | 高 | 启动自动填充 |
| `AsyncStorage` 明文 | 简单存储 | 低 | 不建议 |
| 记忆最近 N 个 Key | SecureStore + 列表 | 高 | 多账号用户友好 |

**推荐**：使用 `expo-secure-store`。
- 写入：`SecureStore.setItemAsync('deepseek_api_key', apiKey)`。
- 读取：AppProvider mount 时 `SecureStore.getItemAsync`。
- 提供“清除 Key”按钮。
- 文案更新为“安全存储在设备钥匙串中”。

### 4.3 历史会话（Scene History）

#### 需求拆解

用户希望：
1. 保存练过的 **scene 文本**。
2. 保存 **对话 segments**。
3. 保存 **练习进度**（currentWordIndex / corrections）。
4. 下次可“继续练习”或“重新开始”。

#### 数据模型建议

```ts
interface PracticeSession {
  id: string;
  scene: string;
  segments: DialogueSegment[];
  currentWordIndex: number;
  corrections: Correction[];
  createdAt: number;
  updatedAt: number;
  title?: string;       // 可自动生成，如 "Coffee run"
  isFavorite: boolean;
}
```

#### 存储策略

| 数据 | 存储位置 | 原因 |
|------|---------|------|
| `apiKey` | `expo-secure-store` | 敏感 |
| `sessions[]` | `AsyncStorage` / SQLite | 量大、可序列化 |
| 当前会话状态 | 内存 Context + 退出时持久化 | 实时性 |

#### UI 建议

- 首页底部或顶部增加 **Recent Sessions** 横向列表。
- 每个会话卡片显示：标题、进度百分比、最后练习时间、继续按钮。
- 支持长按删除 / 收藏。
- 设置页增加“清除所有历史”。

### 4.4 当前高亮单词音频播放

#### 需求

- 用户不知道当前词怎么读，希望听到标准发音。
- 不要太明显，避免干扰练习节奏。

#### 实现方案

**方案 A：底部迷你发音条**
- 在教练面板底部，当前 cue 旁边加一个 🔊 小图标。
- 点击后调用 `expo-speech` 朗读当前单词。
- 图标默认 0.5 透明度，hover/press 时变亮。

**方案 B：单词气泡内长按菜单**
- 长按单词时，除了 Dictionary，再加“Listen”。
- 更符合“查词”心智模型。

**方案 C：自动预读（可选）**
- 当前词高亮时自动轻声朗读，语速可关。
- 适合跟读模式，但可能干扰用户自己发音。

**推荐**：方案 A + B 组合。
- 默认提供低调图标。
- 长按词提供 Listen / Dictionary 选项。

**技术实现**：
```ts
import * as Speech from 'expo-speech';

function speakWord(word: string, language: 'en-US') {
  Speech.speak(word, {
    language,
    rate: 0.9,
    pitch: 1.0,
  });
}
```

### 4.5 中英双语支持（i18n / L10n）

#### 需求场景

- 中国用户居多时，界面用中文更亲切。
- 但练习内容仍是英文。
- 语言切换不应影响 AI 生成语言。

#### 推荐实现

使用 `i18next` + `react-i18next`，或轻量自定义 `I18nContext`。

```ts
// src/i18n/index.ts
export const resources = {
  en: { translation: { ... } },
  zh: { translation: { ... } },
};

// 关键文案全部抽离
{
  "home.title": "Build a speaking drill",
  "home.title_zh": "开始一次口语练习",
  "home.subtitle": "Write the moment, connect DeepSeek, then rehearse with live cues.",
  "home.subtitle_zh": "描述场景，连接 DeepSeek，跟随实时提示练习。",
  "teleprompter.start": "Start speaking",
  "teleprompter.start_zh": "开始练习",
  ...
}
```

#### 设计注意

1. **文案长度差异**：中文标题更短，需避免布局塌陷。
2. **字体**：中文需补充 `"PingFang SC"`、`"Noto Sans SC"` 等回退字体。
3. **AI 提示词**：界面语言切换时，`generateDialogue` 的 `scene` 可以是中文，但 prompt 要明确让 AI 返回英文对话。
4. **日期/数字格式**：历史时间显示用相对时间（“3 分钟前” / "3 min ago"）。
5. **方向**：仅 LTR，无需 RTL。

#### 最小可行方案

若不想引入 `i18next`，可先用 `I18nContext` + `useI18n`：
- 支持 `en` / `zh`。
- 文案集中放在 `src/i18n/strings.ts`。
- 设置页加语言切换开关。

## 5. 核心功能交互再设计

### 5.1 首页重构建议

当前首页是“三步向导”，适合首次使用。但老用户每次进来都要点三下才能开始，效率低。

**建议采用“双模式首页”**：

```
┌─────────────────────────────┐
│  AI scene coach              │
│  Build a speaking drill      │
├─────────────────────────────┤
│  [ 描述新场景输入框 ]         │
│  快捷场景: Coffee / Hotel ...│
├─────────────────────────────┤
│  📚 继续练习                 │
│  ┌─────┐ ┌─────┐ ┌─────┐   │
│  │会话1│ │会话2│ │ +更多 │   │
│  └─────┘ └─────┘ └─────┘   │
├─────────────────────────────┤
│  ⚙️ Settings                 │
└─────────────────────────────┘
```

- 新用户：引导输入场景。
- 老用户：一眼看到最近会话，一键继续。

### 5.2 练习页重构建议

#### 当前问题

1. 控制按钮在底部，但长对话时手指需要跨越屏幕。
2. 教练面板折叠态信息太少，展开态信息过载。
3. 错题提示只显示最近 3 个，缺少完整错题本入口。

#### 建议

1. **手势操作**：
   - 单击词：跳转。
   - 长按词：弹出操作菜单（Listen / Dictionary / Restart from here）。
   - 左右滑动气泡：快速标记为“已掌握”或“重练”。

2. **音频辅助**：
   - 当前词旁边常驻小喇叭。
   - 首次进入时提示“点击小喇叭听发音”。

3. **练习模式切换**：
   - 跟读模式（Full）：练习所有词。
   - 角色模式（Roleplay）：只练习 user 台词，AI 台词由 AI 语音朗读。
   - 纠错模式（Review）：只练习错过的词。

4. **结束页**：
   - 练习完成后弹出总结：准确率、最弱单词、建议再练。

## 9. 错误处理与反馈

### 9.1 错误分级

| 类型 | 示例 | 用户文案 |
|------|------|---------|
| 网络错误 | fetch 失败 | “网络不太稳定，请检查连接后重试。” |
| 权限错误 | 麦克风被拒 | “需要麦克风权限才能练习。去设置开启？” |
| API 错误 | 401 / 429 | “API Key 无效或额度不足，请检查设置。” |
| 生成空内容 | segments 为空 | “AI 没生成出对话，试试描述更具体一些？” |
| 语音识别错误 | 平台报错 | “麦克风休息一下，正在重试…” |

### 9.2 重试机制

- API 失败：指数退避重试 2 次，仍失败则显示手动重试按钮。
- 语音识别失败：自动重启（已实现）。
- 自动续写失败：仅提示，不影响当前练习。

## 10. 性能与 API 成本

### 10.1 Token 成本

- 每次生成约 10-14 行对话，加上 system prompt 和历史对话。
- 自动续写会携带全部历史 segments，随会话变长成本线性上升。

### 10.2 优化建议

| 措施 | 效果 |
|------|------|
| 自动续写只携带最近 6-8 段 | 降低 prompt 长度 |
| 用户可关闭自动续写 | 控制成本 |
| 提供“生成摘要”模式 | 用更短对话练习 |
| 本地缓存相同 scene 的结果 | 减少重复调用 |

### 10.3 渲染性能

- `allWords` 在 `segments` 变化时重建，长会话会重新计算。
- 建议对 `TeleprompterDisplay` 使用 `React.memo`，并按 segment 拆分 memo。
- 自动滚屏的 `useEffect` 依赖较多，可用 `requestAnimationFrame` 节流。

## 11. 隐私与安全

1. **API Key**：必须加密存储，永不提交到 Git。
2. **语音数据**：`expo-speech-recognition` 本地识别或云端识别，需在隐私政策中说明。
3. **历史会话**：用户对话文本保存在本地，不上传。
4. **导出功能**：如支持导出练习报告，需明确告知用户。
